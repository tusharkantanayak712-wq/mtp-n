mlbbtopup.in API v1.0

Official Service API Documentation — For Members & Owners Only

1. Authentication

Every request must include your API key in the request header. Your key starts with TK_ and is shown only once when created. Keep it secret — treat it like a password.
Required Header for ALL Requests
x-api-key: TK_your_api_key_here
Content-Type: application/json

2. Security Layers

3-Layer Protection:
  • IP Whitelisting — Restrict your key to specific server IPs via the Dashboard. Requests from other IPs are instantly rejected.
  • Daily Spend Limit — Automated daily cap (default ₹25,000). Resets every midnight.
  • Rate Limiting — 2-second cooldown between requests to prevent loops.

3. API Endpoints

GET /api/service/balance
Returns your wallet balance, user profile, and current API key usage stats.
Sample Request
GET https://mlbbtopup.in/api/service/balance
x-api-key: TK_your_api_key_here
Sample Response (200 OK)
{
  "success": true,
  "user": {
    "userId": "TK1001",
    "name": "John Doe",
    "userType": "member",
    "wallet": 4850.00
  },
  "apiKey": {
    "name": "My Website",
    "dailyLimit": 25000,
    "usedToday": 150
  }
}

POST /api/service/validate
Validates a player's game account before placing an order. Returns the player's in-game name.
Sample Request
POST https://mlbbtopup.in/api/service/validate
x-api-key: TK_your_api_key_here

{
  "gameSlug": "mobile-legends988",
  "playerId": "12345678",
  "zoneId": "1234"
}
Sample Response (200 OK)
{
  "success": true,
  "playerName": "ProGamer123",
  "playerId": "12345678"
}
POST /api/service/order
Places an order. Funds are instantly deducted from your wallet and the top-up is auto-executed in real-time. No manual processing required.
Sample Request
POST https://mlbbtopup.in/api/service/order
x-api-key: TK_your_api_key_here

{
  "gameSlug": "mobile-legends988",   // from /games list
  "itemSlug": "weekly-pass816",     // from /games/{slug} items
  "playerId": "12345678",
  "zoneId": "1234"              // required for ML, optional for others
}
Sample Response — Success (200 OK)
{
  "success": true,
  "message": "Order fulfilled successfully",
  "order": {
    "orderId": "API-XK9P2A1B",
    "itemName": "Weekly Diamond Pass",
    "price": 165,
    "status": "success",
    "topupStatus": "success"
  }
}
Sample Response — Fulfillment Failed (200 OK)
{
  "success": false,
  "message": "Payment deducted, but fulfillment failed",
  "order": {
    "orderId": "API-XK9P2A1B",
    "status": "failed",
    "topupStatus": "failed"
  }
}

4. Error Codes

HTTP Status Reason How to Fix
401 Missing or invalid API key Check your x-api-key header. Ensure the key is active.
401 IP not whitelisted Add your server IP to the Allowed IPs list in the Dashboard.
403 Daily spend limit reached Increase your daily cap in the Dashboard or wait until midnight reset.
403 Rate limit — too fast Wait 2 seconds between consecutive requests.
403 Insufficient wallet balance Top up your wallet before placing orders.
404 Invalid game or item slug Use GET /api/service/games/{slug} to get valid slugs.
500 Internal server error Retry after a few seconds. Contact support if it persists.

5. Recommended Integration Flow

Step-by-step for building your reseller website:
  1. Call GET /games → Show game list to your customer
  2. Call GET /games/{slug} → Show items & your prices
  3. Call POST /validate → Verify player ID before payment
  4. Collect payment from your customer on your site
  5. Call POST /order → Auto-executes top-up instantly
  6. Call GET /status/{orderId} → Show delivery confirmation
